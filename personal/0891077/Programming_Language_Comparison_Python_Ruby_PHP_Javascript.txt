각 언어의 장점 5가지 정리 및 비교

Python

장점

1
일단 교수님이 보내주신URL에서 간단하게 배우기만 했을뿐인데도 가장 큰 장점이 확 보였습니다.
무엇보다도 print 함수만 봐도 쉽다는것입니다.
비교해주신 다른언어와 비교해봐도 딱 배우기쉬울것 같다는 생각이 들었습니다.

2
두번째 장점으로는 첫번째 장점과 연동되어 문법이 쉬운편이기 때문에 개발기간이 짧다는것입니다.
간단하고 쉬운 문법으로 인해서 비교적 짧은 코드로 인해서 개발기간을 단축할수 있습니다.

3
세번째 장점은 독립적 플랫폼이라 응용성과 확장성또한 매우 뛰어나다는 점입니다.
거의 모든 환경에서 작동할수있다는 장점은 정말 큰 장점입니다.

4
메모리가 자동관리 된다는점도 큰 장점입니다. 쓰래기 수집이라는 기능을 사용해서 필요할떄 메모리를 자동할당하고 모든걸 자동으로 한다고합니다.

5
다양한 자료형(라이브러리)을 제공해서 사용자로 하여금 더 편한 프로그래밍을 해주도록 도와줍니다.



Ruby 

장점

1
자료를 조사하다보니 가장 큰 장점으로 꼽는것이 문법의 자율성이라고 했습니다.
사실 코드를 보면 바로 와닿지는 않았지만 객체지향이든 절차지향이든 뭘로 만들어도 어색하지 않다고합니다.

2
두번째로 강력한 장점은 메타 프로그래밍이라는 점인데 이건 기계적인 코딩을 자동화 시키는 기법이라고 합니다.
이렇게 기계적으로 반복되는 코드를 자동으로 생성하는 기법을 지원한다고 합니다.
이건 개발자에게 지루함을 줄여준다고 생각합니다.

3
파이썬과 비슷한 부분이긴한데 배우기에 쉬운편이라고 합니다. 완전히 다 배우기전에도 활용성도 뛰어나기 때문에
차차 실력을 늘려나갈수도 있어 확장시키면서 활용하기 좋다고 합니다.

4
또 주목받는점 중 하나가 순수 객체지향적 언어라는 것이었습니다. 모든것이 객체로 사용되어 문법이 어렵지않습니다.

5.
마지막으로 이식성 또한높아 모든 플랫폼에서 잘 작동합니다.



PHP

장점

1
탄생 기반 자체가 개인홈페이지 제작을 목적으로한 스크립트 언어에서 발달한 케이스라고합니다.
그렇기 때문에 사이트 제작시에 유용한 함수화가 잘 되어있습니다.

2
파이썬이나 루비와 같이 변수에 타입이 따로 없는 동적언어라서 확장이 자유롭습니다.

3
C언어와 유사한 편이라 C를 배운 개발자들은 확실히 다른 언어보다 조금더 배우기 쉬운편입니다.
또 속도도 C처럼 빠른편입니다.

4
mysql과의 연동이 쉬워서 데이터베이스를 관리함에 있어서 유용하고 웹문서를 수정하는데 용이합니다.

5
루비와 마찬가지로 이식성이 매우 높아 독립적 개발 활용이 가능합니다.



Javascript

장점

1
요즘 대세라고 할수있는 자바의 기본적인 문법 부분을 많이 사용하고있습니다. 
그렇기때문에 자바를 배운 개발자들은 확실히 더 빠르게 배울수 있게 됩니다.

2
이식성이 뛰어나서 자바를 지원하는 다양한 플랫폼, 운영체제(리눅스,윈도우 등등)에서 잘 작동합니다.
이 장점은 특정 환경에 구애받지 않고 다양한 개발이 가능합니다.

3
웹브라우저에서 바로 실행할수 있기 때문에(별도의 컴파일과정이 없기 때문에) 별도의 프로그램 없이 어떤 메모장등 어느 프로그램에서나 저장하고 실행해서 테스트해볼수가 있습니다.
즉 시스템이 가지는 부담 또한 적습니다.

4
웹브라우저 안에서 실행되기때문에 별도의 메모리관리가 따로 필요하지않습니다.

5
많은 개발자들이 사용하고 있어 관련 자료를 쉽게 얻을수 있고 HTML 기반이라 소스를 쉽게 공유하고 가져다가 사용할수있습니다.
