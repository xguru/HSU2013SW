- Python
요즘 웹에서 주목받는 언어다. 가장 큰 장점은 배우기 쉽다는 것이다. C나 Java보다 상대적으로
배우기 쉽다. 또 무료다. 물론 다른 코드들도 무료인 경우가 많지만 어찌됐든 무료다. 
들여쓰기의 의무화로 코드를 한 눈에 직관적으로 알아볼 수 있다는 것도 큰 장점이다.
또한 Linux나 윈도우에서 상관없이 쓰이는 크로스 플랫폼이다. 뿐만, 아니라 구글엔진에서
사용되는 웹 언어 중 하나이다(다른 하나는 java). 잠깐이나마 써 본 내가 느낀 가장 큰
장점은 정말 쉽다는 것이다. 교수님께서도 짧은 코드들, 특히 홈페이지와 관련하여 간단히 무언가를
만들 때 많이 사용하신다고도 하셨는데, 많이 공감이 갔다.

- Ruby
인터프리팅 스크립트 언어이다. 심플한 구조로 쉬운 사용법을 제시하는 언어이기에 초보자도 
쉽게 사용할 수 있다. 가비지 콜렉션이 있어서 메모리를 적절히 관리하고 해제하기 때문에 
메모리 관리에 신경을 쓰지 않아도 되어 편리하다. 프로그램 자체가 매우 간결하여 코딩으로
쉽게 프로그램을 작성할 수 있기 때문에 높은 생산성을 가진다. 또 Code Sense 기능을 탑재하고 있어
개발자들이 굳이 긴 메서드의 이름들을 모두 타이핑 할 필요가 없다는 것도 큰 장점이다.

- PHP
서버 사이드 스크립트 언어이다. PHP의 가장 큰 장점은 빠르다는 것이다. 또한 가볍다. 따로 내장된 커오
라이브러리를 가지고 있기 때문에 훨씬 코딩이 가볍다. 또한 단순 반복적인 코딩이 쉽다. 실제로 구구단을
만들어 페이지를 만들어봤는데 다른 언어보다 쉽고 빠르게 작성할 수 있었다. 코딩을 재활용할 수 있다.
실제로 해보지는 못했지만 복잡한 코딩을 피하여 기본직인 기능은 다른 프로젝트에서 사용한 동일한 코드를
재활용할 수 있다. 또 보안 수준도 높을뿐 아니라 날짜, 데이터베이스 연결 등에서 다른 언어들과는 다른
독창적인 기능이 있어 편리하다.
 
- Javascript
사실 부끄럽게도 교수님께서 말씀해주시기 전까지 Javascript와 Java와 상관이 없다는 사실을 이번에
조사하면서 처음 알게 되었다. Javascript는 주로 브라우저에서의 사용이 목적이다. 이 언어의 장점은
타입이 없다는 것이다. 다른 언어처럼 int, char 이런것 필요없이 사용될 때 그 목적에 따라 알아서
타입이 정해진다. 두 번째는 JSON(JavaScript Object Notaion)이다. 속성을 데이터에 관계없이 바로바로
정의해줄 수 있다. 또한 가장 큰 장점은 직접 결과를 바로바로 확인할 수 있다는 것이다. 뿐만 아니라
HTML문서사이에 직접 코딩이 가능하다는 점도 큰 장점이다.  

<총평>
사실 위의 언어들을 제대로 써보진 못했고, C나 Java처럼 익숙치도 않기 때문에 이 언어들을 
잘 안다고는 못하겠다. 하지만, 이번 기회에 프로그램 언어에는 C나 Java 말고도 참 많은 언어들이
있고, 이 언어들이 각각의 특징들이 있으며 필요성에 따라 언제든 쓰일 수 있다는 점들을
알 수 있었다.
