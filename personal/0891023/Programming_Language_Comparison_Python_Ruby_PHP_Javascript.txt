4개의 프로그래밍 언어 비교

1. Python

파이썬은 컴퓨터식 사고방식으로 하지 않아도 그대로 생각하는대로 표현할 수 있게 해주는 언어다.
게다가 직관적으로 표현이 가능해서 쉽게 알아 볼 수 있다.
가장 큰 예는 다음과 같다. "if 4 in [1,2,3,4]: print"4가 있습니다."
직관적으로 표현이 가능하기 때문에, 문법도 매우 쉬워 빠른 학습이 가능하다. 프로그래밍 경험이 있다면 1주일이면 충분히 배울 수 있다고 한다.
문법이 쉬워서 개발속도도 무척 빠르고 재미있게 프로그래밍을 할 수 있다.
복잡한 연산 등을 제외하면 파이썬으로 불가능한 것은 없다고 한다.
게다가 파이썬은 오픈소스라서 언제 어디서든 사용이 가능하다.


2. Ruby

순수한 객체지향언어라서 원시형 데이터타입을 전혀 지원하지 않는다. 그래서 모든 데이터가 곧 객체다.
'블록'이란 기능을 이용하여 집합 데이터의 원소를 순차적으로 처리하는 작업 등에 유용하다.
파이썬과 같이 코드를 보다 간결하고 직관적으로 만들어준다.
Ruby의 가장 강력한 기능은 메타프로그래밍이다. 메타프로그래밍이란 프로그램 코드를 작성하는 프로그램을 만드는 것인데, 반복적인 코딩을 자동화하는데 매우 유용한 기법이다.
정규식/비정규표현식을 지원하는 Perl의 텍스트 처리 기능을 그대로 계승해서, 텍스트처리가 매우 뛰어난 프로그래밍 언어다.
또한, 배우기 무척 쉬운 언어다. Syntax가 매우 직관적일 뿐만 아니라 완전히 마스터하기 이전에 활용성이 비교적 뛰어난 언어다.


3. PHP

C언어를 기반으로 만들어진 서버에서 실행되는 스크립트 언어로 C언어에 익숙한 개발자들이 보다 쉽게 접근할 수 있다.
데이터베이스 연동을 편리하게 구현할 수 있어서 MySQL, Oracle 및 윈도우 ODBC등 DB를 편리하게 연동할 수 있다.
리눅스, 윈도우 등의 모든 운영체제에서 구현 가능해, 프로그래밍을 하는데 제약이 없다.
코드작성이 쉽고, C/C++문법과 비슷하여 클래스를 지원해 효율적인 코딩이 가능하고, 스크립트 언어이기때문에 HTML문서 내부에 입력이 가능한 언어다.
인터프리터 방식을 채택하고 있어 ASP나 Perl 보다 처리속도가 빠르다.


4. Javascript

자바처럼 자료형, 사용함수를 굳이 알고 있지 않아도, 작성 과정이 매우 쉽고, Html 파일에 추가하여 홈페이지를 역동적으로 만들수 있게 사용할 수 있는 스크립트 언어다.
그래서 자바보다 쉬운 스크립트 언어 때문인지, 쉽게 배울 수 있다.
또한 C/C++같은 다른 언어에 비해 컴파일 과정이 없어 쉽게 결과를 보고 접근할 수 있다.
윈도우 같은 운영체제에 관계없이 웹서버만 있으면 작동할 수 있다.
웹에서 실행되는 만큼 처리속도도 상대적으로 빠르다.


5. 결론

4개 언어 모두 각각 다른 특징이 있지만, 주된 공통점은 배우기 쉽고, 직관적이여서 보는 사람의 입장에서도 보기 편하고, 처리속도도 매우 빠르다는 것이다.
3가지의 장점이 모두 들어갔기때문에 아직까지도 계속 사용되며, 유행에 따라 사용되지 않는 때도 있지만, 결국 다시 사용하게 되는 언어인 것 같다.
개인적으로 자바스크립트의 경우는 자바와 지난학기 서버프로그래밍을 하면서 계속 써봤고, 홈페이지를 개인적으로 만들때 제로보드를 이용하면서
PHP 파일을 고쳐보려고 잠깐 본 경험이 있어서, 4개의 언어중 파이썬을 한번 새로 배워보고 싶다.
