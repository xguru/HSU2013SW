PHP
PHP는 HTML과 연동이 된다. HTML 파일에 <?  ?>같은 기호 사이에 파이썬 코드를 넣기만 하면 HTML이 하나의 코드로 생각해서 연동이된다.
mysql과 쉽게 연동가능하며 다양한 OS를 지원한다.
오픈소스이기 때문에 여기저기서 긁어 올 수 있다. 하지만 오픈소스는 그만큼 취약하므로 조심해야한다.
학원을 다닐 때 리눅스 강의에서 홈페이지 만드는 것을 해본적이있는데 그때 php란걸 한번 써보았다.
비록 명령어를 따라치는 것에 불과 했지만 매우 쉬웠던 걸로 기억한다.
echo 문장만 몇 개 끼워넣어봤는데 형식도 C언어랑 비슷했고 그리 어렵지 않던 언어인걸로 기억한다.
명령어만 익힌다면 제일 간편하게 홈페이지를 만들수 있는 것 같다.

Python
파이썬은 첫 시간부터 교수님께서 맨날 말씀하셨다. 인터넷상에서 꾀나 관심을 끌고 있는 언어이다.
파이썬은 대부분 인터넷(웹)에서만 쓰며 어플만들기에는 사용을 잘 안하는 편이다. 구글엡엔진에서도 파이썬을 지원한다고 한다.
사용법을 찾아보니 언어가 알아보기 쉽게 되어있었다. 이런걸 직관성이라 불르는 듯 한데 c와 비슷해서 그런지 알아보기가 정말 쉬웠다.
또한 특이한 점이 괄호나 세미콜론이 없다.
괄호는 들여쓰기가 대신하는데 들여쓰기 정도로 문단?을 구분한다.
보통 c언어를 보기좋게 쓰기위해 들여쓰기를 하는데 파이썬은 이 것이 필수라서 알아보기가 정말 쉽다고한다.
이 때문에 프로그렘을 쓴 사람이 아니어도 이해하기가 더 편하며 따라서 코드 공유까지 편리하다.
평소에 프로그레밍을 할때 세미콜론 하나 때문에 오류검출이 힘들었을 때가 있었는데 이 언어는 그럴 필요도 없다.(세미콜론사용안함)
그래서 그런지 개발시간도 짧고 완성되었을 때 코드도 다른언어보다 매우 짧다고한다. 때문에 메모리 소모도 적다.
변수의 타잎이 정의되지않아도 변수에 입력되는 값에 따라 타잎이 결정된다고 한다.
제일 중요하다고 생각한건 프로그렘 다짜고 컴파일이 되는게 아닌 한줄씩 컴파일 후 실행을 하기 때문에 오류가 적어질 것 같다.
교수님이 설명해주신 테스트함수기능이 떠올랐다. 매우 유용한 언어라고 생각한다.

Javascript
우선 자바와 자바스크립트는 다른 언어이다.
자바의 웹기술은 서버 쪽을 담당하며 자바스크립트는 클라이언트 쪽을 담당한다.
이 때문에 사용자가 정보를 요청하면 빠르게 전달해준다고 한다.
하지만 연구자료에 의하면 자바를 잘하는 사람은 자바스크립트도 잘한다고 한다.
자바스크립트 역시 오픈소스이다.
파이썬과 같이 변수의 형식을 지정해주는 것이 필요없다.
주요한 기능으로는 다른 응용프로그램 안에 있는 객체도 접근 가능하다는 것이다.
html5 과제하는데 빠지지 않았던 자바스크립트. html5가 뜨면서 동시에 인기가 더욱 상승할 것으로 예측된다고한다.
자바스크립트도 한번 사용해 보았는데 이 역시 c와 매우 비슷했다.
처음에 구조가 약간 특이했지만 익히고 나면 자바와 다른 느낌이 없을 것 같다.
철저한 객체지향언어이기에 유지보수가 쉽다.

Ruby루비
루비 또한 객체지향언어라고 한다. 
처음들어보는 언어인데 사용하기가 매우 쉽지만 배우기가 다른 언어에 비해 어렵다고 한다.
또 이것으로 트위터도 만들어졌다고한다.
내장 기능이 강력하고 모듈이 다양하다고한다.(파이선보다 라이브러리 기능이 강력하다?)
반복적인 개발,컴파일,디버깅 등의 작업을 보다 편하게 할 수 있어서 테스트용으로 빠르게 만들기가 적합할 듯 하다.
루비의 모든 타잎은 클래스라고한다. 따라서 클래스 안에 몇 가지 함수를 포함하고 있다.
찾아본바로는 함수가 매우 유용함.
문자열 조작하는 함수를 보았는데 매우 편리하다.
문자열 비교하는 것만으로도 여러가지 기능이 있으며(어디까지 비교할 것인지, 숫자를 빼고 비교할지 등등)언어 또한 이해하기 쉽게 되어있다.
하지만 못들어본 언어인만큼 인지도나 사용범위가 좁다고한다.
