0891048 홍찬기

Python, Ruby, PHP, Javascript 비교

-Python
우선 파이썬은 컴파일러 언어가 아닌 스크립트 언어라 그런지 처음 접하는데도 읽기 쉽고 사용하기 쉬운 편이였습니다. 특히 각 라인에 대한 결과가 바로 출력되서 특별히 오류검사를 행해야 하는 번거러움이 없었습니다.
파이썬의 또 다른 장점은 간단한 문법과 깔끔한 구문입니다. 특별히 언어에 대한 지식이 없어도 있는 그대로 해석이 가능하기 때문에 이해하기 쉬웠습니다.
기존 언어와 달리 데이터 형 선언도 필요없고 코드의 길이도 짧아지기 때문에 프로그래밍에 대한 부담이 없는 편이였습니다. 이 점이 사용자가 쉽에 파이썬을 접할수 있는 요소인 듯 합니다.

-Ruby
Ruby는 탑입명을 기술하지 않고 전역변수인지, 지역변수인지, 인스턴스변수인지 만을 판별하면 뒵니다. 타입은 런타임 과정 중에 알아서 식별 가능한 언어이기 때문에 타입명을 기술할 필요가 없다는 것이 사용자에게 더 쉽게 다가 올 수 있는 것 같습니다.
중요 특징중 하나는 런타임 과정에서 내장 클래스 변경이 가능하다는 것입니다. 이것은 사용자에게 폭넓은 자유도를 부여하기도 합니다.
그리고 java와 같이 가비지 컬렉터를 기본적으로 제공하기 때문에 사용자에게 편리함을 주는 장점이 있습니다.

-PHP
php는 많은 사용자를 보유하고 있는 오픈 소스 스크립트 언어로 알려져 있습니다. 여러 운영체제와는 관계없이 개발이 가능하고 동일한 코드로 동작합니다. 누구나 사용할 수 있고 학습하기가 쉬우며 실행이 빠르다는 장점이 있습니다.
확장성과 데이터베이스 접근성이 매우 뛰어나다는 장점도 있습니다. 서버에서 동작할 때 스크립트를 처리하는 성능이 뛰어나며 다양한 데이터베이스 관리시스템도 지원합니다.
짧은 시간에 프로그래밍을 학습하는데 큰 장점을 갖는 것 같습니다.

-Javascript
 Javascript의 장점은 서버가 없어도 실행이 가능하다는 점입니다. PHP같은 경우에는 언어를 번역할 서버가 필요하지만 javascript는 그 자체가 브라우져에서 쉽게 해석될 수 있기 때문에, 그저 파일로 지정해 놓으면 브라우져에서 열어도 상관이 없습니다.
 Javascript 역시 배우기 쉽다는 장점이 있습니다. 웹 페이지를 동적으로 만드는데 필요한 기능만 사용이 가능하고 미리 만들어진 소스를 가져다 쉽게 응용이 가능하다는 점이 있습니다.
또한 Javascript의 큰 장점은 이식성입니다. 플랫좀에 의존하지 않고 작동하기 때문에 자바스크립트를 지원하는 브라우저만 있으면 언제 어디서든 사용이 가능합니다. 객체지향 언어이기 때문에 재사용이 가능하며, 사용자 컴퓨터상에서 작동하므로 서버에 주는 부담이 적다는 면도 있습니다.

-실행 속도 비교
Javascript > PHP > Ruby > Python
자바스크립트가 가장 빠르고 파이썬이 가장 느리다.
