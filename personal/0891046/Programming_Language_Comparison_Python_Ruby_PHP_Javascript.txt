Python
프로그래밍 언어 중 한가지로 들여쓰기를 무조건 써야 하도록 만들어 두었다. 그래서 코드를 짜지 않은 다른 사람이 보더라도 코드 
이해가 빠르고 알아보기가 쉽다. PHP와 마찬가지로 주로 웹에서 쓰인다. 구글에서 채택한 3개의 공식언어에도 들 정도로 사용이 많이
 되는 언어이다. 하지만 어플을 만들 때에는 많이 사용 하지 않는 모습을 보인다. 파이썬은 첫 번째 언어부터 객체지향 적으로 만들어
 진 언어이다. 배우기도 쉽고 사용 방법도 쉬우며 간결하게 사용이 가능하다. 자신이 생각한 알고리즘을 프로그램 내에서 거의 
흡사하게 만드는 것이 가능하다. 또한 라이브러리를 사용할 수 있어서 c언어에서 좋았다고 생각하는 라이브러리를 가져다가 쓰는 것도
 가능하다. 문법이 간단하고 객체지향언어를 사용하기 때문에 최신 트렌드를 잘 반영한 언어이다.
Ruby
루비언어 안에는 가비지 콜렉션이라는 프로그램이 내장되어 있어서 메모리를 프로그램 스스로 적절하게 관리한다. 그래서 사용자가
 프로그램내에서 메모리 관리에 고민하지 않아도 되도록 만들었다. 텍스트 관련 처리가 탁월한 언어이다. 그래서 예외 처리와 같은 
것을 보다 알기 쉽도록 프로그램을 작성 할 수 있다. 미리 컴파일 하지 않고 프로그램이 실행될 때 자신이 작성한 코드를 기계어로 
번역하는 인터프리터 언어를 사용한다. 또한 루비는 다른 프로그램에 비하여 배우기 쉽다고 많이 알려져 있다. 간결하고 정확한 
언어를 사용함으로써 생산성을 높였고, 필요하지 않은 코드는 줄이고, 코드 작성을 자동화함으로서 개발 시간을 단축 하는 것이 
가능하다.
PHP
PHP의 장점이라하면 바로 HTML과 직접 연동이 가능하다는 것이다. HTML파일에 바로 PHP코드가 사용이 가능하게 만들 수 있다. 
그래서 HTML 코드 속에 PHP 코드르 끼워 넣는 것도 가능하다. PHP는 크로스 프렛폼이라서 데이터베이스와의 연동이 다른 프로그램에
 비해 쉽다. 또한 문법이 간결하고 명령을 쉽게 전달하는 형식이라서 빠르게 코딩이 가능하다. 또한 예외처리도 C언어와 비슷하게
 구현함으로서 학습에 용이하다. 또한 PHP 프로그램의 속도가 다른 프로그램에 비해 빠르다. 또한 작업환경에도 구애받지 않는다.
 리눅스, 윈도우 등 다른 운영체제에서도 사용이 가능하다. 
Javascript
다른 언어들과 다르게 컴파일 과정이 없다. 그래서 쉽게 접근이 가능하다. 또한 운영체제에 구애받지 않는다. 또한 바로 HTML 문서와
 연동이 되어서 PHP와 비슷한 성향을 보인다. 또한 이번에 자바스크립트는 V8 자바스크립트라는 것을 확장하여, 보다 더 빠른 
서버언어가 되었다. 그래서 속도도 많이 빨라진 것 같다. 또한 웹페이지를 만들 때 보다 더 동적으로 만드는 것이 가능하다. 
또한 객체 지향적이라서 보다 더 고급언어를 사용할 수 있다. 또한 클라이언트 내에서 실행이 가능하기 때문에 서버로 보내고 다시
 받는 것을 하지 않더라도 확인이 바로 가능하다. 그래서 서버부하를 줄일 수 있다는 부과적인 효과도 나온다. 그래서 PHP와는
 다르게 서버 측 언어관련 지연현상이 일어나지 않는다.
