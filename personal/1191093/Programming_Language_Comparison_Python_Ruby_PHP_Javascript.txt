주제 : Python, Ruby, PHP, Javascript 비교 (각 언어의 장점을 5가지씩 정리 및 비교)

1. Python
  - Python은 문법이 마치 영어를 읽듯이 되어 있어 초보자들도 쉽게 배울 수 있으며, 이해 또한 빠르게 할 수 있다. 
  - 다른 언어에 비해 문법이 간결하다
  - 간결한 문법으로 인해 생산성이 높다 (Java,C,C++에 비해 삼분의 일에 해당하는 코드량과 개발시간을 자랑하고있다)
  - Python의 또 다른 장점중에 하나는 오픈 소스여서 따로 사용료 없이 사용이 가능하다는 점이다.  
  - Python은 확장성이 강하다.
  -  하이브리드 객체지향 지원
  실제로 Python에 대해 조사를 진행을 하면서 간단한 예제 코드들을 따라서 실습을 해보았는데, 
  a = 3, b = 3 print a is b를 치니 true라는 결과 값이 나와서 놀랐다. c나 java에 익숙해져서 인지 마치 컴퓨터와 대화를 하는 듯한 느낌을 
  받게되었다. 

2. Ruby
 - 순수 객체지향 지원한다.
 - 유연한 언어이다. (개발자가 원하는 대로 자유롭게 바꿀 수 있다.)
 - 단일 상속만 가능하나 클래스에 모듈을 믹스인 할 수 있다.
 - 변수 선업없이 사용할 수 있다. (몇가지 관례만을 사용한다 )
 - 이식성이 매우 높으며, OS가 지원한다면 확장 기능을 동적으로 load할 수 있다.
 Try Ruby를 통해서 체험을한 Ruby는 약간 Shell 같은 느낌도 있었다. 다른 언어들에 비해서 개인적으로는 이질감 같은게 느껴지기도 했다. 
 그러나 위의 장점처럼 문법이 간결하기 때문에 이해가 어렵진 않았다.
 
3. PHP
 - 운영체제나 플랫폼에 구애 받지 않고 동작 (환경에 독립적)
 - 객체지향 지원
 - 저렴한 비용 (무료 라이센스)
 - 데이터베이스를 편리하게 연동할 수 있다.
 - 처리 속도가 빠르다.
 
 사용을 해본 결과 JSTL과 JQuery를 섞어 놓은듯한 느낌이었다.

4. Javascript
 - 운영체제의 구애 받지 않고 동작(환경에 독립적)
 - 컴파일의 과정이 없이 바로 실행할 수 있어 편리하게 접근할 수있다.
 - 클라이언트에서 실행이 되므로 즉시 확인이 가능하다. (서버의 부하를 줄일 수 있다)
 - 객체 지향적 속성
 - 작성 과정, 문법이 쉽다.
 사용해본결과 Javascript의 작성과정, 문법이 쉬운것이 장점이나 위의 다른 언어들에 비해서는 쉽지 않았다.
