-HTML 5 란 무엇인가?
우선 HTML 5는 HTML에서 부터 발전되어온 기술의 형태이다. 그렇다면 HTML 5의 근간이되는 HTML은 무엇일까? 
HTML, Hypertext Markup Language. 우선 나는 HTML의 뒷 글자,Markup Language(마크업 언어)가 무엇인가에 대해 먼저 알아보기로 했다.
인터넷 검색과 참고서적을 뒤져본 결과 마크어 언어란 웹페이지의 구조를 설명하는 언어란 걸 알 수 있었다. 
즉, 우리가 인터넷브라우저를 통해 접속하면 만나게 되는 웹페이지의 구조를 만드는 언어라고 정의할 수 있겠다.
그렇다면 HTML의 앞글자 HT는 무엇일까? 작년 이맘때쯤 조세홍 교수님께서 멀티미디어 개론시간에
"책을 보면 한시간안에 지루함을 느끼는데 반해 왜 인터넷을 하면 시간가는 줄 모르는 줄 아니 바로 하이퍼 텍스트 기능 때문이야" 라는 말씀을 하셨다.
지금와서 HT와 ML을 비교해 보니 단일페이지의 마크업 언어에서 벗어나 다른 페이지와 연결할 수 있게 해주는 기능이 바로 하이퍼 텍스트라고 생각하였다.
결국 HTML 란 웹상에서 프로그래밍적인 부분은 자바스크립트가 스타일리쉬한 화면은 CSS가 구조는 HTML이 담당하는 기술이라고 할 수 있겠다.
HTML 에서 HTML 5로 버전업이 되면서 이런 단순한 기능 뿐만 아니라 여러 새로운 패러다임을 제시하는 기술이 있을 것인데 그것은 아래서 살펴보겠다. 
 

-HTML 5는 왜 출현했는가?
앞서 말했듯이 HTML 5는 HTML의 다음 버전이라고 할 수 있다. HTML이 발전하면서 웹브라우저 업체들이 자기 브라우저에서만 지원하는 여러 HTML 과 부가기능을 설치하기 시작했다.(지난주 과제였던 IE,크롬,Firefox등이)그 때문에 개발자들은 사용하는 브라우저에 맞는 코드를 작성해야하는 불편함을 겪었고 그뒤로 표준을 제정하기 위한 W3C 의 XHTML 등과의 대립 등 여러 이야기가 이어진다. 하지만 결국 HTML을 끌어안고 몇가지 발전된 기능을 가진 HTML 5가 표준이 되었다. 앞서 얘기 되었던 여러 플랫폼사이의 호환문제를 여러 공개 진행과정을 통해 개발자의 의견을 수렴한 것이 핵심이라 할 수 있겠다.
이 부분은 리눅스가 나오게된 오픈소스 운동과도 밀접한 관련이 있는 듯 하다.


-HTML 5의 현재 상태는?
2000년대 마이크로 소프트의 IE는 사실상 W3C의 웹표준이 된다. 하지만 위에서 살펴보았던 HTML 5가 XHTML을 제치고 표준으로 자리잡게 되면서 HTML 5 의 기능을 빠르게 받아드리지 못한 IE의 쇠퇴가 시작된다. 현재에 이르러 Chrome,Firefox등 다양한 웹브라우저가 HTML 5의 기술을 토대로 IE의 아성에 도전하고 있고 마이크로 소프트 또한 HTML 5의 기술을 빠르게 흡수하기 위해 윈도우 8을 발표하는 둥 발버둥 치고 있다. 개방형,웹 생태계의 시대가 도래하면서 이에 최적화된 HTML 5 기술은 각광받고 있다.


-HTML 5의 미래는?
미래의 트랜드중 하나가 HTML 5라는 사실은 누가 들어도 자명한 사실이다. 그렇다면 왜 HTML5인가? HTML 5가 기반을 두고 있는 웹 환경은 발전 영역이 무궁무진하다. 더욱이 스마트폰시대로 들어서면서 모바일 시장 또한 열리게 되었다. 하지만 여러 디바이스에서 HTML 5가 사용된다면 역시나 호환성 문제가 대두되기 마련이다. 그래서 이런 호환성 문제를 없애기 위해 HTML 5는 웹에서 직접 어플리케이션을 만든다. 즉 미래에는 모바일에서도 데스크탑에서도 HTML 5에 의한 개발이 많아질 것이다. 또한 구글 크롬북과 마이크로 소프트의 윈도 8의 기본 내장 어플리케이션이 모두 HTML 5로 개발되었다니 이젠 바야흐로 HTML 5의 시대라 할 수 있겠다.


-HTML 5의 주요기능은?
HTML의 주요기능 중은 거의 자바스크립트 기반의 API 로 구성되어 있다.그중 관심있거나 신기한 기능들에 대해 찾아보았다.

지오로케이션: HTML 5의 지오로케이션 기능은 브라우저를 통해 우리가 어디에 있는지를 알 수 있게 해준다.(위도와 경도로)
나의 위치정보를 브라우저를 통해 알 수 있는 것은 단순한 위치정보 뿐만을 이야기 하는 것이 아니다.
나의 위치를 통해서 방향을 알려주고, 목적지까지의 거리를 알려주고, 심지어 가까운 거리에 있는 공통의 관심사를 가진 사람들을 연결해줄 수도 있다. 

로컬 저장소: HTML5는 데이터를 클라이언트에 저장할 수 있는 기능을 제공한다. 기존에 사용하던 쿠키의 개념이 아니라(더 발전하여) 웹페이지의 키/값 쌍을 로컬 저장소에 저장할 수 있는 기술이다. 크롬을 쓰면서 내가 방문했던 사이트를 어플리케이션화 시킬 수 있었던 것도,웹페이지가 하나의 OS가 된다는 개념도 모두 이 기술을 바탕으로 나오는 이야기 인 것 같다.

캔버스:HTML이 HTML 5로 진화하면서 더 이상 마크업언어에만 머물지 않게 되었다. 바야흐로 프리젠테이션의 시대를 맞아 우리가 사용하는 IT 시장에도 픽셀을 손쉽게 다룰수 있는 방법이 도래 한 것이다.
이 기술이 브라우저를 통해 실현된 것이 바로 캔버스이다. 모바일 장치나 테스크탑 모두에서 HTML 5의 캔버스 기술은 구현이 가능하기 때문에 우리는 웹페이지에서 모바일 어플리케이션을 통해 원하는 형태의 그림을 그리고 서로 공유할 수 있다. 이는 어느 기기에서나 내가 원하는 그림을 그릴 수 있고 이를 동적으로 조정할 수 있다는 의미인데 디지털 영상처리나 컴퓨터 그래픽스를 배우는 우리 과에도 밀접한 관련이 있는 듯 하다.


-HTML 5의 브라우저 지원상태는?
HTML, CSS 그리고 HTML 5로 들어가면서 추가된 자바스크립트가 실행되는 공간은 웹브라우저이다. 그러니 웹브라우저 지원 상태는 굉장히 중요한 부분이겠다. 
자신이 쓰는 웹브라우저로 홈페이지를 접속하면 HTML 5의 TEST SCORE 를 제공해주는 사이트도 있어서 들어가 봤더니 윈도우 7에 CHROME을 쓰는 내 점수는 500점 만점에 468 점이였다.
그 외에 타임라인 형식으로 HTML 5 점수를 매겨놓은 표를 보니 Chrome 과 firefox가 IE 에 비해 훨씬 높은 점수를 받고 있다. 
구체적인 주요기능으로 보더라도 캔버스,websockets등 IE는 Chrome 이나 firefox 등에 비해 지원이 늦거나 대체로 지원이 없다. 교수님께서 왜 지난주 과제에 웹브라우저에 대해 조사해보고 장단점을 비교하라고 했는지 왜 이젠 IE를 버리라고 했는지 알 것 같다.
