Python

모든 프로그램은 데이터를 처리하는 데 궁극적인 의미가 있다고 할 수 있다. 우리가 2학년때 배운 자료구조도 '데이터를 어떻게 컴퓨터 상에 저장할 것인가' 에 대한 학문이고 알고리즘 또한 '자료를 어떻

게 로드하는 것이 적합한가'에 대해 연구하는 과목이라 생각한다. 
이런 의미에서 파이썬은 데이터를 다루기 위한 리스트를 쉽게 만들고 효과적으로 처리 할 수 있다.
예를 들어 id=["kim yong jin",0891008,"multimedia"] 식으로 리스트를 만들 수 있다. 기존의 언어들과 다른 점이 두 가지 있었다.
첫째,우리가 c언어나 c++ ,자바를 배우면서 선언했던 int id[10] 같은 일련의 변수형을 선언하지 않고 매우 간단하다는 것.
둘째, 타입을 선언하지 않은 만큼 id 라는 리스트에 숫자든 문자열이든 모두 들어 갈 수 있다는 것.
이 두가지를 고려해 봤을때 리스트 선언을 매우 빠르고 가볍게 할 수 있었다.
소프트 웨어 공학 수업을 들으면서 항상 드는 생각이 있다. 개발자는 열려있어야 하는 구나. 시대의 흐름은 '오픈된' 세상이구나.
파이썬의 특별한 기능 중에 함수를 공유할 수 있는 모듈과 배포 유틸리티 프로그램을 지원한다.
실습을 통해 함수를 배포해 보지는 못했고, 함수를 .py 모듈로 변환한 후 PyPI에 등록하는 과정까지 쭉 따라가 보았다.
그 외에도 스크립트 언어의 특성상 다른 프로젝트에 통합하기가 쉽다는 장점. 루프를 사용하지 않고 복잡한 데이터구조를 짧게 처리할수 있는 특징. 등 
이번 방학에 꼭 한번 다시 접해봐야하는 언어임을 확실히 깨달았다.

do print "Hello, world!" end 같은 코드를 실행 시키면 Hello, World! 를 10번 찍어준다. 마치 c언어나 java 처럼 어떤 정형화된 문법이 있는 것이 아니라 그보다는 더 유연한 우리가 쓰는 언어 같은 느낌

PHP

PHP의 가장 큰 장점은 Apache 와 MySQL 과의 호환성이다. Apache + PHP + MySQL을 합쳐서 'APM'이라고 부르기도 하는데 이들 모두는 공개 소프트웨어이고 무료이다. 무료라고 성능이 떨어지는 것은 절대 아

니기 때문에 쇼핑몰과 같은 소규모 웹사이트를 구축하는데 많이 사용된다. 앞선 과제로 html에 대해 알아봤었다. html5와는 좀 다른 얘기지만 html은 정적인 언어이고 웹페이지를 보여주는 용도로 밖에 작

성할 수 없기 때문에 그를 보완하기 위해 PHP가 나왔다.
PHP는 웹페이지가 클라이언트의 브라우저에 전송되기 전에 서버를 통해 페이지의 내용을 조작할 수 있게 한다. 즉 서버와 브라우저 사이에 PHP가 관여하여 웹 어플리케이션을 마음대로 변화시킬 수 있게 한

다. 동적인 HTML이 되도록 하는 것이다. 그래서 대부분의 PHP 스크립트는 PHP 코드와 HTML 코드 모두를 포함하고 있다. 언어의 형식도 HTML과 많은 점에서 닮아있다. 
현대로 오면서 단순한 보여주기 페이지만이 필요치 않게 되면서 PHP를 통해 데이터베이스에서 꺼내온 데이터를 보여주고 메일 전송을 바로 해주는 등의 기술이 필요하게 되면서 발전하게 되었다.

Javascript

javascript는 웹상에서 사용자의 반응에 동적으로 반응하기 때문에 생기발랄한 역동성을 위해 필요하다. 이전 주 과제,html5 에 대해 조사하면서 html은 웹상의 뼈대를 만들고 css는 시각적인 효과를 만든

다고 조사했었다. 거기에 javascript는 페이지가 어떤동작을 수행하도록 기능을 넣는다. 예를들어 alert()라는 함수는 사용자에게 정보를 보여주기 위해 사용하는 팝업박스를 이야기한다. 이 언어는 다른 

타언어와는 다르게 웹상에서 바로 실행될 수 있는 조건을 갖췄다. 실습으로 크롬에서 ctrl + shift +j 를 누르고 console 창에 alert('hello world'); 란 자바스크립트 코드를 쳐보니 경고창이 바로 튀어나

왔다.alert는 하나의 함수일뿐 javascript 또한 java같은 언어와 같이 재사용이 가능한 코드를 여러개 가지고 있었다.
또한 javascript는 if문과 switch/case문,for문 등 우리가 전에 배웠던 조건문도 쓸 수 가 있다. 자바와 자바스크립트는 어떤 하등의 관계가 없다는게 정설이지만 마치 내 느낌으로는 Java 언어가 웹에 생

태계에 맞게 변화한게 Javascript인 것 같다. java의 플랫폼에 독립적인 가장 큰 장점 역시 가지고 있으니 말이다.
이번 과제를 통해서 Javascript의 아주 간단한 '쉽고', '빠르고', '동적으로 동작한다'는 기본적인 요소만을 알았다. 결국 문제는 이런 언어를 언제 심도 있게 접하느냐 인데 교수님 말씀대로 학교에서 알

려주는 언어 뿐만 아니라 새로운 언어 또한 유연하게 다룰 줄 알도록 노력해야겠다.
