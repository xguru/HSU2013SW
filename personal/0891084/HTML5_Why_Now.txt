HTML5 란 무엇인가?
일단 HTML은 Hypertext Markup Language의 약자로서 뜻을 살펴보면,
일반 문서는 사용자와의 인터랙션이 없이 계속 일정하게 정보를 순차적으로 주지만,하이퍼텍스트(Hypertext)는 사용자와 인터랙션을 가지며 원하는 순서와 원하는 정보를 얻을수잇는 시스템이다.이러한 인터랙션은 노드끼리의 네트워크를 통하여 정보를 얻을수있다(노드:하이퍼텍스트의 기초적인 정보단위) 마크업언어(Markup Language)는 홈페이지에 보이는 것을 동적으로 움직이는 것.문서에 포함된 문장, 표, 소리 등과 같은 문서의 내용에 대한 정보가 아니라 그 문장과 그림, 표가 어떻게 배치되고 글자는 어떤 크기와 모양을 기지며, 들여쓰기와 줄간격, 여백은 어느 정도인지에 대한 정보를 가짐 따라서 HTML은 웹페이지를 사람들이 좀더 편리하게 사용할수있도록 도와준다. 과거의 HTML과 달리 HTML5는 플렛폼에 구애를 받지않는다는 것이다. 기존의 HTML은 스마트폰이나 브라우저에따 일일이 개발을 해야했으나 HTML5는 이를 보안하여 멀티플랫폼이 가능하다.
기존의 HTML은 동영상재생이나 그래픽구현,정보입출력을 위해서는 activeX나 플래쉬,실버라이트,java 등 별도의 설치가 필요했다, 하지만 HTML5는 이를 호환할수있는 힘이 있다. 따라서 어떤 스마트 디바이스로 접속하건 어떠한 브라우저로 접속하건 동일한 정보와 동일한 화면을 제공받을수있게 된다는 것이다. IT산업의 발전에 따라 플래폼전략이 중요한데 HTML5는이러한 전략에 도움이될 것 이다.
현제는 모바일 디바이스의 웹활동이 가파르게 증가하는데, ABI리서치는 2016년 HTML5를 지원하는 브라우저를 탑재한 스마트폰 추이를 21억대가량으로 잡고 있는것만 보아도 HTML5의 힘과 중요성을 알수 있다. HTML5의 특징과 주요기능들이 있는데 다음과 같다.
웹폼 (Web Form)-사용자의 입력정보를 받기 위해 사용되는 입력 형태에 대한 정의에 사용되는 마크업.  
캔버스 (Canvas)-웹에서 즉시모드(immediate mode)로 2차원 그래픽을 기리기 위한 API와 canvas 내 각종 객체를 회전, 변환하고 그레디언트, 이미지 생성 등 각종 효과를 주는 기능에 대한 API 지원  
SVG (Scalable Vector Graphic)-XML 기반의 2차원 벡터 그래픽을 표현하기 위한 언어  
Video/Audio-video는 비디오 또는 영화를 보여주기 위해 사용되는 미디어 엘리먼트이며, audio는 사운드나 오디오 스트림을 표현하기 위한 미디어 엘리먼트&Geolocation-디바이스의 지리적 위치 정보를 제공하는 API 표준 지원 
오프라인 웹 응용 (Offline Web Application)-인터넷 연결이 지원되지 않는 경우에도 웹 응용이 정상적으로 수될 수 있도록 지원하는 기능으로 응용에 대한 캐싱과 데이터에 대한 캐싱으로 구성  
웹 SQL 데이터베이스 (Web SQL Database)-다양한 표준 SQL을 사용해 질의할 수 있는 데이터베이스 기능에 대한 API 지원  
로컬저장소 (Local Storage)-기존의 쿠키의 기능을 개선하기 위한 목적으로 개발된 기능으로 웹 클라이언트에서 키와 값이 쌍으로 구성된 데이터를 영구적으로 저장하는 기능
웹 소캣 (Web Socket)-웹 응용이 서버 측의 프로세스와 직접적인 양방향 통신을 위한 API 지원  - 웹 워커 (Web worker) : 웹 응용을 위한 쓰레드(Thread)-기능에 대한 API 지원  이와같은 기능을 구현하는 HTML5는 HTML4보다 확장된 태그를 지원한다. 또 문서 구조에 적합하게 구조화 마크업을 사용함으로써 개발자를 위한 것이 잘 나타나 있다. 
현재 브라우저들의 지원상태와 현황은 http://html5test.com 에서 볼수있다. 홈페이지를 보면 현재 브라우저들의 스코어를(보급상태) 확인할수 있다. 현시점엔 Chrome 26 ≫Maxthon 4.0 ≫Opera 12.10 ≫Firefox 20 ≫Safari 6.0 ≫Internet Explorer 10 순으로 지원율이 좋다.
