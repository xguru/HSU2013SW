0891007 김영수 소공 프로그래밍 언어 비교
교생실습 중이라서 모든 언어를 써 볼 시간이 없었었습니다. 그래서 인터넷에서 검색하고 제가 이해한대로 작성하였습니다. 나중에 하나하나 다 사용하여 보도록 하겠습니다.

Python은 귀도 반 로섬이라는 프로그래머가 개발한 고급 프로그래밍 언어입니다. 이 Python 이라는 이름은 귀도 반 로섬이 좋아하는 코미디 이름에 파이선이 들어가서 여기서 따온 것이라고 합니다. Python의 장점으로는 첫째로 코드의 이해도가 빨라서 초보자가 배우기 쉽습니다. 두 번째로 c로 1000줄이 넘는 코드도 Python으로는 100줄로 단축 할 수 있어서 개발 시간을 단축시킬 수 있다고 합니다. 셋째로 사용범위가 매우 넓어서 여러 분야에서 많이 사용되고 있습니다.  넷째로 표준 라이브러리를 가지고 있고 많은 라이브러리를 제공합니다. 다섯째로 Ruby와는 다르게 하이브리드 객체 지향을 지원한다는 것입니다. C보다 속도가 느리다는 단점이 있지만 간단한 프로그램의 경우에는 느끼기 어렵다고 합니다.

Ruby는 마츠모토 유키히로라는 일본 사람이 개발한 동적 객체 지향 스크립트 프로그래밍 언어입니다. Python과 비교했을 때 세계적으로는 Python 사용자가 많지만 일본사람이 만든 Ruby는 일본에서 인기가 많습니다. Ruby의 장점으로 첫째로 유연한 언어라는 것입니다. Python은 정형화된 요구를 하는 반면 Ruby는 정형화된 서식을 요구하지 않습니다. 개발자가 어느 부분이든지 수정할 수 있습니다. 둘째로 Ruby는 이식성이 높아서 유닉스나 맥, 윈도우시리즈 등을 가리지 않고 동작합니다. 셋째로 쉬운 프로그래밍입니다. 개발자 마츠모토 유키히로의 철학인 ‘Ruby는 스트레스가 없는 쉬운 프로그래밍이고 프로그래밍은 컴퓨터가 아닌 인간 중심으로 설계되어야 한다’가 잘 반영되어 있습니다. 넷째로 프로그래밍 처리속도가 빠르고 메모리 사용량이 적다는 장점이 있습니다. 그리하여 빠르게 웹 어플리케이션을 만들 수 있습니다. 다섯째로 Python과는 다르게 순수 객체 지향을 지원한다는 장점이 있습니다.

PHP는 Personal Hypertext Preprecessor의 약자로서 프로그래밍 언어의 일종입니다. 장점으로는 첫째로 HTML과 연동이 된다는 것입니다. 홈페이지를 만들 때 PHP로 만들었던 게 기억이 납니다. 둘째로 무료라는 점입니다. 따라서 낮은 비용으로 좋은 결과를 낼 수 있습니다. 세 번째로 기능을 접하기 쉬워서 배우기 쉽고 가볍고 빠르다는 점이 있습니다. 넷째로 기존까지 존재해 왔던 언어에 비해 직관적 이라는 장점이 있습니다. C언어에서 접근하려면 여러 Include를 통해야 했던 작업들을 그냥 적어놓기만 하면 작동을 합니다. 그래서 코드의 양이 현저히 줄어들게 됩니다. 다섯 번째로 PHP는 지루한 단순 반복 작업을 할 필요 없이 쉽게 목적을 달성할 수 있습니다.

JavasScript는 객체 기반의 스크립트 프로그래밍 언어로 Java와는 관련이 없습니다. JavaScript의 장점으로는 첫째로 Html과 연동이 좋아서 Html만으로는 만들기 힘든 홈페이지를 html의 코드에 추가하면 만들 수 있습니다. 둘째로 javascript는 윈도우나 유닉스 등의 운영체제에 관계없이 잘 작동한다는 점입니다. 셋째로 클라이언트에서 실행이 되므로 서버의 부담을 줄여줍니다. 넷째로 다른 언어에 비해 컴파일하는 과정이 없어서 접근성이 쉽다는 장점이 있습니다. 다섯 번째로 다른 언어에 비해 명령어의 수가 많지 않은 편이고 문법도 쉬워서 작성과정이 매우 쉽다는 장점이 있습니다.
