-Python-

1. 파이썬의 문법은 간결하고 가독성이 좋다. 특히 코드블럭을 들여쓰기로 구분하는데 이것은 가독성이 높아질 수밖에 없는 구조이다.

2. 파이썬에는 매우 광범위한 라이브러리가 기본으로 포함되어 있으며, 확장성도 무궁무진하다.

3. 기본적으로 제공되는 라이브러리 말고도 쉽게 라이브러리를 추가할 수 있는데, C로 구현되어 있는 모듈을 쉽게 만들어 붙일 수 있다.
특히 파이썬은 C보다는 느리므로 속도 문제가 생기는 부분을 C로 구현해 붙일 때 유용하다. 또한 파이썬에는 없고 C에는 이미 있는 기능을 
붙이는 데도 쓰일 수 있다. 반대로 파이썬의 기능을 C에서 사용할 수 있다.

4. 파이썬에서는 문자열이 모두 유니코드를 나타낸다. 즉, 우리가 한글, 한자 등을 표현하기 위해 별도의 노력을 기울이지 않아도 된다는 것을
의미한다.

5. 파이썬은 런타임 시에 타입 체크를 하는 동적 타이핑과 자동으로 메모리 관리를 해주는 특징이 있다.


-Ruby-

1. 루비에서는 메타프로그래밍을 지원하기 떄문에 런타임에 프로그램코드를 동적으로 추가할 수 있다.

2. 루비는 Perl의 텍스트 처리 기능을 계승하고 있는데, 이 때문에 루비는 텍스트처리에 매우 뛰어나다.

3. 루비는 순수 객체지향 언어이기 때문에, 루비의 모든 데이터는 객체로 이루어져 있다.

4. 루비는 해석형 언어이기 떄문에 실행속도가 다소 느리다.

5. 루비만의 강력한 기술이 인정되어 많은 애플리케이션과 프레임워크가 루비를 기반으로 만들어졌고,
루비 온 레일스가 그 예로 들 수 있다.


-PHP-

1. 코드작성이 간다하고 쉽다. Perl이나 C, Java와 같은 다른 CGI와 달리 PHP는 HTML을 출력하는데 많은 명령어가 필요 없다. HTML내에 하고자 하는 일에
대한 스크립트를 적어주면 된다.

2. PHP는 데이터베이스를 폭넓게 지원하고 있다. mSQL,MySQL,Oracle,PostresSQL 등이 있는데  PHP는 어떤 웹프로그램과는 비교도 못할 만큼 DB연결이 쉽다.

3. PHP는 유닉스계열에서 Apache 웹서버에 모듈 형식으로 붙을 수도 있고 Windows 환경에서 IIS나 PWS 또는 Ppache 웹서버와 함께 사용 가능하다.
또한 Unix환경에서 작성한 코드를 수정 없이 Windows 환경에서 사용 가능하다.

4. PHP 언어의 문법은 C언어와 매우 유사하기 떄문에 C언어에 익숙한 사용자는 PHP를 쉽게 적응 할 수 있다.

5. PHP는 무료이고, 배우기 쉽고, 웹싸이트를 만드는데 유용하고,확장성이 좋다.


-Javascript-

1. 자바 스크립트는 인터프리터 방식의 언어이다. 인터프리터 방식의 언어는 별도의 컴파일 과정을 거치지 않고 소스 코드를 인터프리터가 읽어 바로 실행하는
방식으로 동작한다. 인터프리터 방식은 컴파일러 방식에 비해 수행 속도가 느린 문제점읋 가지고 있지만 컴파일러 방식의 언어보다 단순하여 배우기 쉬우며
개발과 수정, 디버깅이 간편하고 편리한 장점 또한 가지고 있다.

2. 자바 스크립트는 HTML 페이지 내에 삽입되어 클라이언트 브라우저에서 실행되므로 웹 페이지 상의 날짜 형식을 자동으로 변경시킨다거나 연결된 페이지가
팝업 윈도우 내에 나타나게 한다거나 마우스의 움직임에 따라 텍스트나 그래픽 이미지를 변화시키는 등의 일을 할 수 있다.

3. 자바 스크립트는 C++에서 파생된 컴파일러용 객체지향 언어인 자바의 개념을 일부 사용하므로 자바와 비슷한 면도 있지만 다른 면도 많이 가지고 있다.

4. 자바스크립트는 windows, UNIX 등 어떤 운영체제에서도 작동하는 이식성이 좋은 언어이다.

5. 자바스크립트는 HTML에 삽인되어 전달되므로 네트워크 연결등이 요구되지 않으므로 시스템 부하가 적다.
