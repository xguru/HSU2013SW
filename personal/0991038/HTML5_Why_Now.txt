1. HTML5가 무엇이고 왜 출현했는가?

HTML은 1991년에 처음 사용하기 시작한 웹페이지 제작용 마크업 언어입니다. 1995년 W3C에서 정식으로 HTML 2.0 표준안을 발표한 후 
보완과 업데이트 과정을 걸쳐 1999년에 HTML 4.01, 2001년에 HTML을 XML형태로 사용할 수 있도록 개선한 XHTML이 발표되었습니다.
그 후 2005년, 오페라,모질라,사파리와 같은 웹브라우저 업체들은 XML기반의 XHTML을 보완해 표준으로 제정하려고 했던 W3C와 별개로
WHATWG이라는 표준화 기구를 만들어 Web Application 1.0이라는 이름으로 별도의 표준안 제정을 진행했습니다. W3C의 XHTML 2.0은
실제로 사용하기 복잡하고 현실성이 떨어진다는 평가가 내려졌고 이와는 반대로 WHATWG의 Web Application 1.0은 진행 과정에서 여러
다양한 개발자와 웹브라우저 업체의 의견을 받아들여 유연한 표준안으로 업그레이드 되었습니다. 결국, W3C는 2007년 XHTML2.0을
포기하고 WHATWG의 표준안을 받아들여 HTML5라는 이름으로 표준안을 내놓기로 결정하였습니다. HTML5는 리치 컨텐츠를 만들기 위해
사용되는 여러가지 관련 기술들을 포함하는 포괄적 용어이기도 합니다.

2. HTML5의 현재 상태와 미래는?

핵심 HTML5 표준은 아직까지 개발 중에 있습니다.  HTML5 표준화는 2014년 최종 표준 제정을 목표로 작업 중입니다.
유명 기업들은 HTML5를 이용해 많은 비용을 절감하고, 다양한 기기에 컨텐츠를 전달하고 있습니다. HTML5 웹 앱은 한번 제작되어지면
거의 모든기기에서 실행이 가능합니다. 현재 80% 가량의 브라우저가 HTML5를 지원하고 있고 그 수치는 점점 증가하고 있습니다.
또한 HTML5는 50% 이상의 개발자들이 활용하고 있으며, 가까운 미래에는 그 비중이 90%로 늘어날 것으로 예상된다. 


3. HTML5주요기능은?

HTML5의 대표적인 주요기능은 기존의 웹페이지 마크업 언어로의 역할 외에도 웹 애플리케이션 개발용 스펙을 포함시켜
멀티미디어, 드래그앤드롭과 같은 기능을 담은 리치 웹 미디어 요소의 역할도 합니다. 다음의 대표적인 HTML5의 핵심 기능입니다.

1. 기존의 HTML, XHTML의 호환성을 유지하면서 유연한 범용 표준안 지원.
2. 간략화된 DTD 선언문(!doctype html) 지원.
3. 웹 문서의 의미 있는 구조화 작업을 위한 header, nav, footer, articl과 같은 시맨틱 요소 지원.
4. 웹 폼에서 다양한 속성 추가.
5. 그래픽 API에 사용할 수 있는 canvas 요소 지원.
6. 비디오 및 오디오 재생을 위한 video, audio 요소 지원.
7. SQL 기반 데이터베이스 지원을 위한 기능 추가.
8. 오프라인 웹 애플리케이션 기반의 API 기능 지원.
9. 드래그앤드롭이 가능한 API 기능 지원.

4. HTML5의 브라우저 지원상태는?

대부분의 유명한 웹 브라우저들은 이미 부분적으로 HTML5를 시행하고 있습니다. 구글 크롬 모질라 파이어폭스 등이 그 예 입니다.
HTML5의 특성을 사용한 많은 사이트들이 빠르게 늘어나고 있습니다. 몇몇은 간단한 실험사이트들로 HTML5의 기능들을 적용했을 때
그 기능들이 어떻게 나타나는지 보여주고 있지만 HTML5의 주요한 기능을 잘 활용하고 있는 사이트들 또한 점차 증가하고 있는 
상황입니다. 유투브를 좋은 예로 들 수 있습니다. 유투브는 현재 HTML5 비디오 지원을 제공하고 있습니다.
하지만 모든 브라우저들이 HTML5의 전 기능을 지원하고 있는 것은 아닙니다. 브라우저가 매번 업데이트 될 때마다 새로운 기능을
보여주고 버그들도 개선되는 것을 볼 수 있습니다. 이 이야기는 어떤 기능이 어떤 브라우저에서 지원되고 있는지에 대한 완벽한
정보를 주는 것은 어렵다는 것을 의미합니다.
