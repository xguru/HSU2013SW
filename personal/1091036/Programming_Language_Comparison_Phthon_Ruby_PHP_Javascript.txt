Python
 파이썬을 깔고 써본 첫 느낌은 일단 언어가 쉬웠다. 내 손에 익지 않는 데서 오는 어색함을 빼고는 매우 편했다. 
사실 언어를 쓰면서 느끼는 것은 실무에서는 모르겠지만 거의 대부분이 for문과 if문이 다라는 것 이였다. 이와 같은 이유로
튜토리얼에서 제일 먼저 for와 if를 봤는데, 간단한 포문을 돌릴때도 일일이 변수를 선언하고 해야 하는 불편함이 파이썬엔
range하나로 해결되었다. 또 거의 C와C++에서의 언어와 비슷하기 때문에 기본적인 언어만 알고 있다면 금방 배울 수 있을 것
같았다. 또 조금은 유행이 간 듯 하지만 가상현실에서도 이용되는 언어로 알고 있다. 이는 플랫폼이 다양하고 컴파일을 하지
않아 빠른 속도를 자랑하는 특징과 관련이 있는 듯 하다. 또한 이클립스에서도 실행 가능 한 것으로 보아 유연한 언어로 보인다.

Ruby
 루비 역시 수업시간에 처음 들어본 언어였다. 설치를 하고 실행이 어려워 이래저래 찾아가니 교수님이 링크 걸어준 곳에서 
체험만 해보게 되었다. 체험해본 느낌은 체험이어서 인지 모르겠지만 정말 쉬웠다. 너무 쉬워서 조금 손해 보는 느낌도 들었던
것 같다. 프로그래머의 편리성을 위하여 만들어졌다는 소개글을 본 것 같은데 맞는 말인 것 같다. 문법이 굉장히 직관적이고
간결하여 가독성이 좋다. 또한 객체지향 언어로 모든 것을 객체로 사용하여 사용법을 쉽게 익힐 수 있다고 한다. 아무것도
모르던 1학년때 봤어도 코드를 이해 할 수 있을 것 같은? 내 생각엔 빠른 시간에 결과물을 만들어낼 수 있다는 점에선 단연
최고일듯싶다

PHP
 PHP를 인터넷에 쳐보니 HTML문서 읽기라는 단어가 많이 보였다. 알고 보니 PHP는 웹 개발을 위해 만들어진 언어라고 한다.
간단히 사용 해봤을 시에도 HTML를 쓰는 느낌이였다. 웹 프로그래밍 공부는 HTML을 아주 약하게 배운 정도라 그런지 이 언어는
많이 생소하였다. 실행은 HTML의 문서 중간에 PHP라는 조건을 단 문장을 집어 넣었더니 PHP처리 기능을 가진 웹페이지가 코드를
읽고 만들어 주었다. PHP는 플랫폼에 제한되지 않아 다양한 플랫폼에서 동작하고 데이터베이스 프로그램과의 연동이 쉽다고
한다. C언어 기반이지만 클래스를 제공하고 포인터가 없으며 변수의 값들은 참조의 형태로 전달하는 객체적인 성격을 띄고 있다.
사용하기 조금 까다로워 사용 예시를 보았는데 있는 내용을 그대로 옮겨 적어서 직관적이라는 느낌이 들었고 복잡해 보였다.

Javascript
 일단 Javascript를 써보고 가장 놀란 점은 자료형이 var 하나라는 것 이였다. 굳이 자료형을 따로 나누지 않으니 그것에서
오는 편함은 이루 말할 수 없었다. 또한 조금 쓰다 보니 java와 전혀 다르다는 것을 알지만 문법이 java와 유사한 것 같았다.
그렇지만 자바보다는 가볍고 빠른 느낌? 내 생각을 말하자면 자바는 프로그래밍언어라면 자바스크립트는 좀더 웹쪽에 맞춰진
느낌이였다. 조사해보니 자바는 HTML의 부족한 부분을 채우기 위해 만들어 졌다고 한다. 또한 응답이 빠른 인터페이스를 만들 
수 있게 해주어 시간 절약이 많이 되며 브라우저 상에서 어떠한 작업들이 가능한지 테스트를 하고 이에 대한 리액션을 바로 
해준다고 한다. 자바스크립트 역시 객체지향 언어이며 웹 상에서 많은 점유율을 가지고 사용되어졌지만 HTML5의 등장과 함께
다시 한번 주목 받아지고 있다고 한다.
