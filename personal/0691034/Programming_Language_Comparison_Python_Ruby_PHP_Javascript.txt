각 언어의 장점을 5가지씩 정리 및 비교. 각300자 이상

1. Python
 1) Python은 플랫폼으로부터 독립적이다. 그래서 윈도우, Linux, Mac 등 대부분의 운영체제에서 실행이 가능하다.
    또한 python은 무료로 제공되고 있기 때문에 학생, 개발자, 개발회사들이 접하기 쉽다.
 2) Python은 컴파일(compile)방식이 아닌 인터프리터(interpreter) 방식이다. 프로그램의 전체를 보고 실행을 하는
    컴파일러와는 달리 인터프린터는 명령을 한줄씩 입력할 때마다 그에 대한 실행을 한다.
    따라서 복잡한 프로그래밍을 하고 있을 때 특정 부분이 잘 실행되는지 확인하기 쉽기 때문에 편리하다.
 3) Python은 프로그래밍 언어에 대한 지식이 뛰어나지 않을지라도 간단한 문법 덕분에
    코드를 이해하고 사용하기 편리하다.
 4) Python은 유니코드를 사용하여 문자열을 구성한다. 따라서 한글과 같은 문자를 사용하기 위해 별도의 작업을
    하지 않아도 되서 편리하다.
 5) Python은 다른 언어들에 비해 문법이 간편하여 코드 수가 적다. 따라서 개발시간을 단축 시킬 수 있다.

2. Ruby
 1) Ruby는 c/c++과 유사한 스크립트 언어라 c를 아는 이에게 어렵지 않다. 
    또한 완전히 무료이기 때문에 사용, 복사, 배포 등을 마음대로 할 수 있다.
 2) Ruby에서는 모든 것이 객체이다. 다른 언어에서와는 다르게 숫자 역시 객체이다. 모든 것이 객체이기 때문에 사용하기 쉽다.
 3) Ruby는 이식성이 높아 Linux, Unix, Mac, Windows, DOS 등에서 동작한다.
 4) Ruby는 스레드 기능을 제공하는데 Ruby가 동작하는 모든 플랫폼에서 가능하다. 스레드 기능을 지원하지 않는
    DOS에서도 스레드 프로그램을 만들 수 있다.
 5) Ruby는 개발자가 원한다면 코어 부분을 포함한 다른 부분들도 제거하고 다시 정의할 수 있을 정도로 
    자유롭게 변경이 가능하다.

3. PHP
 1) PHP는 웹프로그래밍 언어로 Javascript와는 다르게 클라이언트가 아닌 서버에서 실행되는 스크립트 언어이다.
 2) 서버를 구축할 때 비용을 절감할 수 있게끔 해준다. PHP는 리눅스 서버에 쓰이는 언어이기 때문에 공짜인 리눅스 OS,
    Apache, MySQL등을 통해 하드웨어 비용말고는 돈이 들지 않는다. 또한 윈도우 서버에 비해 하드웨어 권장 사양도
    낮기 때문에 더욱 더 하드웨어 비용을 절감할 수 있다.
 3) PHP와 잘 맞는 MySQL을 비롯하여 Oracle, MS-SQL, mSQL 등 다양한 데이터베이스를 지원하며,
    DB처리시에 ASP, JSP에 비해 빠른 속도를 자랑한다.
 4) PHP는 Apache, MySQL 덕분에 Windows, Unix, Linux와 같은 다양한 운영체제에서 사용이 가능하다. 
 5) PHP는 인터프리터 방식의 언어로 컴파일할 필요 없이 바로 실행 가능하다. 따라서 작업 및 수정이 간편하다.

4. Javascript
 1) Javascript는 HTML 문서에 직접 삽입하여 사용이 가능하다. 이를 이용하여 HTML만의 기능으로 할 수 없는
    동작들을 할 수 있게끔 만들어준다. (따로 .js파일을 만들어서 사용하기도 한다.)
 2) Javascript는 인터프리터 방식의 언어로 HTML과 같이 작성하고나서 바로 브라우저에서 볼 수 있다.
 3) Javascript는 다른 언어들이 비해 문법이 쉽고, 명령어의 수 또한 많지 않아 프로그래밍하기 쉽다.
 4) Javascript는 UNIX, Linux, Windows와 같이 많이 사용하는 운영체제에서 제한없이 사용이 가능하다.
 5) Javascript는 클라이언트에서 실행하는 언어이기 때문에 서버가 없어도 브라우저에서 실행가능하다.
