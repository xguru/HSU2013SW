"IE, Chrome, Firefox 사용기"

멀티미디어공학과 0691034 정기백


< Internet Exploror(IE) >
소감 : 우리나라 사람들에게 가장 익숙한 브라우저. 가장 많이 사용하는만큼 이 브라우저를 토대로 웹페이지들이 만들어지는 경우가 많다.
       특히 Active X가 많이 필요한 우리나라 웹페이지 같은 경우에는 IE가 확실히 사용하기 편한 모습을 보인다.
       개인적으로는 IE를 오래 사용해서 단축키도 많이 외우고 있기 때문에 사용하기 편리하다.
장점 : Active X가 많이 설치되어있는 우리나라 웹페이지들을 이용하기 편리하다. 특히 우리나라 각종 결제 시스템엔 IE가
       가장 이용하기 편리하게 되어있다.
단점 : 유저가 잘 모르는 상태에서 필요없는 Active X를 마구잡이로 설치하다보면 브라우저 속도가 급격히 느려진다.


< Chrome >
소감 : Chrome은 웹페이지 로딩시간이 가장 빠르기로 유명하다. 개인적으로도 빠른 속도 때문에 Chrome를 사용하고 있다.
       다만 우리나라 웹페이지들은 Active X를 많이 사용하는데 Chrome은 Active X를 사용하기 위해서는 따로 설정해줘야 하는 것들이
       있기 때문에, 여기서 불편함을 보인다.
장점 : 위에서 말한 것과 마찬가지로 Chrome은 속도가 빠르다. Chrome을 처음 실행시켜서 웹페이지가 뜨는 속도, 
       URL에 주소를 입력해서 사이트가 로딩되는 속도 같이 웹페이지를 여는 속도는 타 브라우저(IE, Firefox)보다 빠르다.
단점 : 탭을 사용해 많은 웹페이지를 띄울 경우 메모리 사용량이 다른 브라우저들에 비해 기하급수적으로 늘어나는 편이다.
       그래서 다른 작업들과 함께 여러 웹페이지를 켜놓을 경우 띄워놓은 웹페이지들이 죽고, Chrome이 다운되는 모습도 종종 보인다.
       따라서 저사양 컴퓨터나 무거운 작업, 고사양 게임 등을 한꺼번에 켜놓고 사용하는 이에게는 불편한 점도 있다.


< Firefox >
소감 : 주변에 프로그래머분들도 Firefox에 대해선 별로 언급이 없어서 이번에 처음으로 사용하게 된 브라우저.
       사용한 기간이 짧아 많은 경험을 하지는 못하였다. 다만 오픈소스 브라우저라 앞으로의 행보가 더 기대되는 브라우저.
       특히 부가기능과 테마변경은 유저들에게 많은 편리함을 주는 것 같다. 예를 들면 필자는 번역 부가 기능을 설치하였는데
       구글번역기를 이용하거나 웹페이지에서 단어를 따로 찾을 필요없이 번역기능을 눌러서 새로운 작은 창에 텍스트를
       복사해서 번역을 누르면 되었기 때문에 간편하였다. 또한 요즘 많은 네티즌들이 이용하는 트위터나 페이스북 같은 SNS 앱도
       있어서 활용성, 확장성에 있어서 다른 브라우저들보다 강점을 보이는 듯 하였다.
장점 : 탭을 사용하여 많은 웹페이지를 열어놓았을 경우 다른 브라우저들에 비해 메모리 사용량이 현저히 적은 모습을 보였다.
       (다른 프로그램들과 함께 다중 작업을 할 경우 컴퓨터에 무리없이 빠른 작업 가능.)
       소감에서도 언급했듯이 다른 브라우저에 없는 각종 부가기능들을 Firefox 브라우저에 설치할 수 있다. 그 종류만해도
       수백가지. 사용자에게 필요한 기능이 있다면 얼마든지 확장시켜 브라우저를 사용할 수 있을 것이다.
단점 : 사진 저장할때마다 다운로드 관리자가 떠서 어떤 파일을 받았는지 확인 가능. 하지만 그로 인해 사진 저장이 다른 브라우저에 비해
       더 느렸다. 웹페이지가 뜨는 도중에 속도가 느린 경우 IE같은 경우는 F5나 주소창 옆의 새로고침 키를 누르면 좀더 빠르게
       웹페이지를 로딩할 수 있으나, Firefox에는 그 기능을 위해선 탭을 우클릭 해서 탭 새로고침을 눌러야해서 익숙하지 않은 사람에겐 
       불편할 듯 하였다.


[ 차이점 ]
'Windows 작업 관리자' 실행시키면 '프로세스'란에서 현재 실행되고 있는 프로그램들의 메모리 사용량을 확인 할 수 있다.
메모리 사용량이 낮아야 다른 여러 프로그램들과 함께 돌릴 때 컴퓨터에 무리를 주지 않고 빠른 속도로 사용할 수 있다.
임의의 같은 웹페이지 1개를 켜놓았을 경우 Chrome이 172,500으로 가장 많은 메모리를 사용하고 있었다.
그 다음은 Firefox(163,600) - IE(89,300)순이었다. IE는 Chrome의 절반정도 밖에 되지 않는 메모리 사용량을 보였다.

그 다음 비교에서는 임의의 동일한 웹페이지 5개를 탭에 실행시켜놓고 메모리 사용량을 측정하였다.
Chrome이 454,400으로 여전히 가장 많은 메모리를 사용하고 있었다.
그 다음은 IE(242,800) - Firefox(195,000)순이었다.
여기서 놀라운 점은 Firefox의 메모리 사용량이었다. 여러개의 웹페이지를 켜놓았음에도 불구하고 메모리 사용량은
다른 브라우저들과는 달리 크게 늘지 않았다. 요즘같이 여러개의 웹페이지를 켜놓고 다른 작업을 동시에 하는
추세에서는 Firefox가 다른 브라우저들에 메모리 사용량이 확연히 적었기 때문에 충분히 사용할만한 가치를 보였다.
