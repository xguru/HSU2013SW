1. HTML5가 무엇인가, 왜 출현했는가?
HTML5는 HTML(Hyper Text Markup Language)이라고 하는 웹 문서 제작 기본 프로그래밍 언어의 최신규격이다.
1999년 HTML4가 표준안으로 제정되었지만, 인터넷의 사용이 일상화되며 사용자들의 요구는 더 늘어났다.
단순히 웹 문서에 글씨나 이미지를 화면에 나타내던 HTML4는 사용자에게 만족을 주지 못하였다.
사용자들은 뛰어난 그래픽과 멀티미디어를 원하였으나 HTML4로서는 불가능하였다.
그리하여 월드 와이드 웹 컨소시엄(W3C)에서 XML과 HTML을 결합한 XHTML을 만들었으나
Google Maps, Ajax, Web 2.0의 등장과 유행으로 웹 애플리케이션이 주목을 끌게 되었다.
그렇게 XHTML은 주목받지 못한 채 사라지게 되었다.

그리고 HTML4로 만족을 얻지 못하는 사용자들의 욕구 충족을 위해 각종 Plug-in들이 생겨나게 되었다.
바로 우리에게 익숙한 플래시(Flash), 액티브X(Active X), 실버라이트(Silverlight), 자바FX(JAVA FX)와 같은 것들이었다.
사용자는 더 뛰어난 그래픽과 멀티미디어의 사용을 위해 이러한 Plug-in들을 설치해야하는 일이 생기게 되었고
몇몇 플랫폼과 브라우저에서는 해당 Plug-in을 지원하지 않아 관련 웹페이지나 웹 애플리케이션을 이용할 수 없는
문제가 생기게 되었다. 이에 W3C를 주축으로 하여 애플, 구글 그리고 모질라 개발자들이 함께 모여 HTML4를 뛰어넘는 
웹 기술 표준화 규약을 만들었는데, 그것이 바로 HTML5이다.


2. 현재상태는? 미래는?
아직 HTML5는 현재진행형이다. 특히 우리나라의 경우 HTML5를 지원하지 않는 IE8이하를 많이 사용하고 있어서
HTML5의 사용은 아직 광범위하지 않다. 또한 아직은 은행권, 공공기관 등에서 사용하는 Active X를 대체할 것이 없다.
Active X의 경우 보안성과 관련되기 때문에 이 문제가 해결되려면 좀 더 시간이 걸릴 듯 하다.

하지만 미래에는 Flash, Silverlight등의 Plug-in을 따로 설치하지 않아도 손쉽게 동영상 시청이 가능할 것이며
각종 Active X를 깔지 않아도 편하게 웹을 즐길 수있을 것이다. 따라서 무분별한 Plug-in, Active X를 설치하지 않아도
되기 때문에 컴퓨터 시스템의 자원을 효과적으로 이용할 수 있을 것이다. 이에 의해 사용자들은 
지금보다 훨씬 쾌적한 환경, 속도에서 웹을 이용하는 것이 가능하게 될 것이다.

3. 주요기능은?
가장 큰 특징은 Canvas 태그이다. Canvas태그를 통해 웹 브라우저에 그래픽을 직접 그릴 수 있게 되었다.
그리고 비디오/오디오 태그에 의해 특별한 Plug-in의 설치없이도 동영상과 음성의 플레이가 가능해졌다.
Local Storage 기능도 생겼는데 이는 사용자의 개인용컴퓨터 저장공간에 DB를 저장하여 네트워크가 연결되지 않은
상태에서도 저장한 웹 메일을 볼 수 있는 것이다.
WebWorker를 이용하여 병렬처리도 가능하게 되었다. JavaScript에서는 쓰레드 제어 명령이 없어 백그라운드에서
동시에 다른 애플리케이션이 처리를 하기 힘들었지만, Webworker를 이용하여 수월하게 처리 가능하게 되었다.
그리고 Geolocation API를 이용해서 위치정보를 알아내고 사용할 수 있다.
또한 새로운 폼 컨트롤들이 추가되었고 위에 언급한 API외에도 다양한 API를 지원한다고 한다.

4. 브라우저 지원상태는?
최근에 업데이트된 대부분의 브라우저들은 HTML5를 지원하고 있다. Chrome, Firefox, Opera, Safari가 앞장서서 HTML5를
지원하였고, IE는 비교적 늦게 HTML5를 지원하기 시작하였다.
Chrome 26, Maxthon 4.0, Opera 12.10, Firefox 20, Safari 6.0, IE 10 순으로 HTML5 지원 테스트 점수가 높은 편이다.
우리나라 한국에서는 아직 많은 사람들이 Windows XP를 사용하고 있는데, 여기에서는 브라우저를 최신으로 업그레이드하여도
IE 8까지 밖에 사용할 수 없다. IE 8 같은 경우는 HTML5의 기능을 대부분 사용할 수 없어 Windows XP와 IE8의 사용 비율은
앞으로 계속 줄어들 전망이다.
간단히 HTML5 지원 테스트 점수를 비교하자면 Chrome 26은 468점, IE10은 320점 그리고 IE8은 42점으로 IE8은 거의 지원하지않고
있다는 것을 볼 수 있다.
